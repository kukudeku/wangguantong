```text

│       │   ├── mapper/       访问数据库

│       │   ├── common/       统一返回结果

│       └── resources/

└── wangguantong-web/         Vue 前端
    ├── package.json          前端依赖和运行命令

    └── src/
        ├── api/              后端接口调用函数
```

可以把整个系统想成一条流水线：

```text
用户点击按钮
  -> Vue 页面中的函数
  -> src/api 中的请求函数
  -> Java Controller
  -> ServiceImpl 业务规则
  -> Mapper
  -> MySQL 数据库
  -> Result 返回 JSON
  -> Vue 更新页面
```

## 3. 推荐阅读顺序

不要按文件夹从头到尾硬看。按下面顺序更容易理解：

1. `database/wangguantong.sql`
   先看系统保存了哪些数据，以及各个状态字段可能出现什么值。
2. `wangguantong-server/src/main/java/.../entity/`
   对照数据库表理解 Java 对象。
3. `wangguantong-server/src/main/java/.../controller/`
   了解系统对外提供哪些接口。
4. `wangguantong-server/src/main/java/.../service/impl/`
   阅读真正的业务规则，重点看实时扣费和预约。
5. `wangguantong-web/src/api/`
   看前端如何调用刚才读过的后端接口。
6. `wangguantong-web/src/views/`
   最后阅读页面模板、数据和按钮事件。

第一次建议只追踪一个简单流程，例如“管理员新增电脑”；理解后再看复杂的“用户实时上机”。

## 4. Java 后端分层

### 4.1 Entity：数据长什么样

以 `Member.java` 为例，它对应 MySQL 的 `member` 表：

```text
数据库 id_card      <-> Java idCard
数据库 member_level <-> Java memberLevel
数据库 create_time  <-> Java createTime
```

`application.yml` 中开启了下划线转驼峰，所以这些字段可以自动对应。

常见注解：

- `@TableName("member")`：这个 Java 类对应 `member` 表。
- `@TableId`：该字段是主键。
- `@TableField(exist = false)`：只用于返回页面，不是数据库真实列。

### 4.2 Mapper：怎样访问数据库

每个 Mapper 都继承 `BaseMapper<实体>`。MyBatis-Plus 已经提供常见 SQL，因此空接口也有用：

```java
mapper.selectById(id);  // 按主键查询
mapper.insert(entity);  // 新增
mapper.updateById(entity); // 按主键修改
mapper.deleteById(id);  // 按主键删除
```

项目更多时候通过 Service 中的 `getById`、`save`、`updateById`、`removeById` 间接使用这些能力。

### 4.3 Service：业务规则放在哪里

Service 接口说明“能做什么”，`service/impl` 说明“具体怎样做”。

例如办理上机不只是新增一条记录，还必须：

1. 检查会员存在且状态正常。
2. 检查会员当前没有其他进行中的上机记录。
3. 检查电脑为空闲状态。
4. 计算会员折扣后的首小时费用。
5. 检查并扣除余额。
6. 新增上机记录。
7. 把电脑状态改为“使用中”。

这些步骤集中在 `OnlineRecordServiceImpl.startOnline()`。

### 4.4 Controller：浏览器怎样找到业务

Controller 把网址映射到 Java 方法。例如：

```text
POST /online/start
  -> OnlineController.start(...)
  -> OnlineRecordService.startOnline(...)
```

常见注解：

- `@RestController`：该类提供 JSON 接口。
- `@RequestMapping("/online")`：该类所有接口共用 `/online` 前缀。
- `@GetMapping`：处理查询请求。
- `@PostMapping`：处理新增或业务操作请求。
- `@PutMapping`：处理修改请求。
- `@DeleteMapping`：处理删除请求。
- `@RequestBody`：从请求 JSON 中读取 Java 对象。
- `@RequestParam`：从网址 `?name=张三` 中读取参数。
- `@PathVariable`：从 `/delete/3` 中读取编号 `3`。

### 4.5 Result：为什么每个接口都有 code/data/message

后端统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

前端 `src/utils/request.js` 会统一检查 `code`。成功时页面只拿到 `data`；失败时统一弹出 `message`。

## 5. Vue 前端基础

一个 `.vue` 文件通常有三部分：

```vue
<template>页面结构</template>
<script setup>数据和交互逻辑</script>
<style scoped>当前页面样式</style>
```

### 5.1 常见写法

- `ref([])`：创建响应式单值；JavaScript 中读写需要 `.value`。
- `reactive({})`：创建响应式对象，适合表单。
- `computed(() => ...)`：根据其他数据自动计算结果。
- `onMounted(fn)`：页面显示后执行，通常用于首次查询数据。
- `onBeforeUnmount(fn)`：离开页面前执行，通常用于清除定时器。
- `v-model="form.name"`：输入框和变量双向同步。
- `v-if="condition"`：条件满足才渲染元素。
- `v-for="item in list"`：按数组重复渲染元素。
- `@click="save"`：点击时执行 `save` 函数。
- `:disabled="condition"`：冒号表示后面是 JavaScript 表达式，而不是普通文字。

### 5.2 API 文件为什么很短

`src/api` 只定义请求地址，例如：

```javascript
export function getComputerList(params) {
  return request.get('/computer/list', { params })
}
```

页面不用关心完整域名、统一错误弹窗和 `Result` 解包，这些都由 `src/utils/request.js` 处理。

### 5.3 登录状态在哪里

本项目把登录对象保存在浏览器 `localStorage`：

```text
admin -> 管理员登录信息
user  -> 普通用户登录信息
```

`src/router/index.js` 的全局路由守卫在每次跳转前检查这两个值。该方式适合课堂项目，但正式生产系统通常应使用后端会话或安全令牌，并且密码必须加密保存。

## 6. 核心业务流程

### 6.1 用户上机与实时扣费

入口：用户端点击机位卡片的“上机”。

```text
terminal.vue submitOnline
-> api/online.js startOnline
-> OnlineController
-> OnlineRecordServiceImpl.startOnline
```

首小时在上机成功时立即扣除。后端每 60 秒执行 `settleRunningRecords()`：

```text
已过分钟数
-> 向上取整得到计费小时
-> 小时单价 × 计费小时 × 会员折扣
-> 减去 totalAmount（以前已经扣过的钱）
-> 只补扣新增差额
```

例如每小时 6 元，普通会员不打折：

```text
开始上机：扣 6 元，totalAmount = 6
第 61 分钟：累计应收 12 元，只再扣 6 元，totalAmount = 12
第 121 分钟：累计应收 18 元，只再扣 6 元，totalAmount = 18
```

余额不够支付新增小时会调用 `autoEndForInsufficientBalance()`，结束记录并释放电脑。

### 6.2 预约电脑

用户只能预约当前时间到未来一小时内的空闲电脑：

```text
新增预约 -> reservation_record 状态“已预约”
         -> computer 状态“预约锁定”
预约上机 -> 复用普通上机扣费流程
取消/超时 -> 预约状态“已取消”并释放电脑
```

后端每 60 秒扫描一次超时预约。用户点击预约上机时还会再次检查时间，避免正好处于两轮定时扫描之间。

### 6.3 购物车与订单

用户端购物车只存在浏览器内存中，结构为“商品编号 -> 数量”。点击“下单”后，页面按购物车清单逐项调用订单接口。

会员订单创建时立即扣余额。后台点击“完成订单”表示商品已交付；点击“取消订单”会把未完成会员订单的金额退回。

### 6.4 充值

本项目不接真实支付接口。用户确认充值后直接调用后台同一个充值接口：

```text
member.balance 增加
+ recharge_record 新增明细
```

这两个操作使用事务，任何一步失败都会一起回滚。散客首次充值后自动转为普通会员。

### 6.5 修改密码和注册

用户登录账号是身份证号。用户注册后默认为普通会员；管理员新增用户时也必须设置密码。修改密码需要提交旧密码和新密码，由后端校验旧密码。

## 7. 状态值速查

电脑状态：

- `空闲`：可上机、可预约。
- `使用中`：已有用户上机。
- `预约锁定`：已被预约。
- `维修中`：不可操作。

上机状态：

- `进行中`：持续计费。
- `已完成`：正常下机或余额不足自动下机。

预约状态：

- `已预约`：等待用户上机。
- `已上机`：已经从预约开始上机。
- `已取消`：用户取消或超时自动取消。

订单状态：

- `已下单`：等待后台制作/交付。
- `已完成`：已经交付。
- `已取消`：已取消，会员订单已退款。

## 8. 配置文件说明

### 8.1 后端 `pom.xml`

- `spring-boot-starter-web`：提供 HTTP 接口、JSON 和内置 Tomcat。
- `mybatis-plus-spring-boot3-starter`：提供 Mapper 和通用增删改查。
- `mysql-connector-j`：让 Java 能连接 MySQL。
- `spring-boot-maven-plugin`：生成可执行 JAR。

### 8.2 前端 `package.json`

JSON 标准不允许注释，直接写 `//` 会导致 `npm` 解析失败，因此该文件保持原样：

- `scripts.dev`：执行 `npm run dev`，启动本地开发服务器。
- `scripts.build`：执行 `npm run build`，生成生产文件到 `dist`。
- `vue`：页面框架。
- `vue-router`：页面路由。
- `axios`：发送 HTTP 请求。
- `@arco-design/web-vue`：后台表单、表格、弹窗等 UI 组件。
- `vite`：前端开发与打包工具。

`package-lock.json` 是 npm 自动生成的精确依赖版本清单，也不应手工加注释或修改。

### 8.3 环境变量

`VITE_API_BASE_URL` 决定前端请求哪个后端：

```text
本地：/api（由 Vite 反向代理到 8087）
宝塔：/api（由 Nginx 反向代理到 8087）
```

## 10. 学习时怎样验证理解

每次选择一项小功能，按以下方式追踪：

1. 在 Vue 模板找到按钮的 `@click`。
2. 在同一文件的 `script setup` 找到对应函数。
3. 看函数调用了哪个 `src/api` 方法。
4. 根据 API 地址找到同名 Java Controller。
5. 继续进入 ServiceImpl，列出它检查和修改的数据。
6. 回到 SQL 文件，确认相关字段和状态确实存在。

建议依次练习：查询电脑 -> 新增电脑 -> 充值 -> 点餐 -> 预约 -> 实时上机。这样复杂度会逐步增加。
